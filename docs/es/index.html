¿Atascado en algún paso?

FAQ

Dudas más comunes en los primeros 15 minutos con el kit.

¿Cuál encuesta debo ejecutar primero?

Para consultoría seria, ejecuta las 3 en este orden: Developer Survey (anónimo) → Learning Survey (identificado) → Assessment principal (liderazgo evalúa informada). Para baseline rápido del board, ejecuta solo el Assessment principal. El agente @ai-maturity-assistant te guía en la elección.

Los comandos /calcular-scores, /gap-analysis no aparecen al escribir /

Copilot Chat necesita estar en modo Agent. Haz clic en el dropdown junto al ícono de Copilot y elige Agent. Si aún no aparece, recarga la ventana de VS Code (Cmd/Ctrl+Shift+P → "Developer: Reload Window").

¿Copilot Free funciona o necesito Pro / Business / Enterprise?

Las skills custom funcionan en Free con límites de mensajes. Para el flujo completo (158 preguntas + 5 PDFs + cross-survey) recomendamos Copilot Pro, Business o Enterprise. Confirma con tu organización si hay restricciones a skills custom en planes enterprise.

¿Hay datos de ejemplo para probar sin llenar todo?

Sí. El kit incluye respostas.json.example (Cliente Exemplo S.A., 46 respuestas simuladas) y respondentes mock para las 2 encuestas. Ejecuta cp respostas.json.example respostas.json y luego /pipeline-completo para ver el output completo en ~3 minutos.

¿Cómo agrego múltiples respondentes del Forms?

La skill /importar-respostas-excel hace promedio automático por pregunta cuando el Excel tiene múltiples filas. Mismo comportamiento que la plataforma web. Mínimo recomendado: 3 respondentes para el assessment principal, 5 para el Developer Survey.

¿Necesito instalar dependencias de Python?

Sí, solo 3: jinja2, weasyprint y openpyxl. Ejecuta make install-deps o pip install jinja2 weasyprint openpyxl. En macOS WeasyPrint además necesita brew install pango; en Linux/WSL, el paquete libpango-1.0-0.

¿Puedo personalizar el branding de los PDFs?

El kit usa la identidad paulasilva-ms (paleta Microsoft 4 colores). Para personalizar: edita relatorios/scripts/branding.py con tus constantes (color primario, autor, contacto, tagline) y re-renderiza. Los 5 templates Jinja2 en relatorios/templates/ consumen estas variables automáticamente.

¿Cómo se integran las 3 encuestas?

Cada output alimenta el siguiente: la Learning Survey genera un plan que el wizard auto-fill (modo D) usa para llenar 6 de los 9 inputs del Implementation Guide. La Developer Survey calcula madurez real por dimensión, cruzable contra la madurez declarada por la liderazgo en el PDF de Score Justification.
