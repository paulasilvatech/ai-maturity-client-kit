FAQ

Dúvidas mais comuns nos primeiros 15 minutos com o kit.

Qual survey devo rodar primeiro?

Para consultoria séria, rode os 3 nessa ordem: Developer Survey (anônimo) → Learning Survey (identificado) → Assessment principal (liderança avalia informada). Para baseline rápido apenas do board, rode só o Assessment principal. O agente @ai-maturity-assistant te guia na escolha.

Os comandos /calcular-scores, /gap-analysis não aparecem ao digitar /

Copilot Chat precisa estar em modo Agent. Clique no dropdown ao lado do ícone do Copilot e escolha Agent. Se ainda não aparecer, recarregue a janela do VS Code (Cmd/Ctrl+Shift+P → "Developer: Reload Window").

Copilot Free funciona ou preciso de Pro / Business / Enterprise?

Skills custom funcionam no Free com limites de mensagens. Para o fluxo completo (158 perguntas + 5 PDFs + cross-survey) recomendamos Copilot Pro, Business ou Enterprise. Confirme com sua org se há restrições de skills custom em planos enterprise.

Tem dados de exemplo para testar sem preencher tudo?

Sim. O kit vem com respostas.json.example (Cliente Exemplo S.A., 46 respostas mockadas) e respondentes mock para os 2 surveys. Rode cp respostas.json.example respostas.json e depois /pipeline-completo para ver o output completo em ~3 minutos.

Como agrego respostas de múltiplos respondentes do Forms?

A skill /importar-respostas-excel faz média automática por questão quando o Excel tem múltiplas linhas. Mesmo comportamento da plataforma web. Mínimo recomendado: 3 respondentes para o assessment principal, 5 para o Developer Survey.

Preciso instalar dependências Python?

Sim, apenas 3: jinja2, weasyprint e openpyxl. Rode make install-deps ou pip install jinja2 weasyprint openpyxl. No macOS o WeasyPrint precisa de brew install pango; no Linux/WSL, do pacote libpango-1.0-0.

Posso customizar o branding dos PDFs?

O kit usa a identidade paulasilva-ms (paleta Microsoft 4 cores). Para customizar: edite relatorios/scripts/branding.py com suas constantes (cor primária, autor, contato, tagline) e re-renderize. Os 5 templates Jinja2 em relatorios/templates/ consomem essas variáveis automaticamente.

Como funciona a integração entre os 3 surveys?

Cada survey output alimenta o próximo: o Learning Survey gera um plano que o wizard auto-fill (modo D) usa para popular 6 dos 9 inputs do Implementation Guide. O Developer Survey calcula maturidade real por dimensão, que pode ser cruzada com a maturidade declarada da liderança no PDF de Score Justification.
