FAQ

Most common questions in the first 15 minutes with the kit.

Which survey should I run first?

For serious consulting, run all 3 in this order: Developer Survey (anonymous) → Learning Survey (identified) → main Assessment (leadership reviews informed). For a quick board baseline, run only the main Assessment. The @ai-maturity-assistant agent guides the choice.

/calcular-scores, /gap-analysis commands don't appear when I type /

Copilot Chat needs to be in Agent mode. Click the dropdown next to the Copilot icon and pick Agent. If it still doesn't show, reload the VS Code window (Cmd/Ctrl+Shift+P → "Developer: Reload Window").

Does Copilot Free work or do I need Pro / Business / Enterprise?

Custom skills work on Free with message limits. For the full flow (158 questions + 5 PDFs + cross-survey) we recommend Copilot Pro, Business or Enterprise. Confirm with your org if there are restrictions on custom skills in enterprise plans.

Are there sample data to test without filling everything?

Yes. The kit ships with respostas.json.example (Cliente Exemplo S.A., 46 mocked responses) and mock respondents for the 2 surveys. Run cp respostas.json.example respostas.json then /pipeline-completo to see the complete output in ~3 minutes.

How do I aggregate multiple Forms respondents?

The /importar-respostas-excel skill does automatic mean per question when the Excel has multiple rows. Same behavior as the web platform. Recommended minimum: 3 respondents for the main assessment, 5 for the Developer Survey.

Do I need to install Python dependencies?

Yes, only 3: jinja2, weasyprint and openpyxl. Run make install-deps or pip install jinja2 weasyprint openpyxl. On macOS WeasyPrint also needs brew install pango; on Linux/WSL, the libpango-1.0-0 package.

Can I customize the PDF branding?

The kit uses the paulasilva-ms identity (Microsoft 4-color palette). To customize: edit relatorios/scripts/branding.py with your constants (primary color, author, contact, tagline) and re-render. The 5 Jinja2 templates in relatorios/templates/ consume these variables automatically.

How do the 3 surveys integrate?

Each survey output feeds the next: the Learning Survey generates a plan that the wizard auto-fill (mode D) uses to populate 6 of the 9 Implementation Guide inputs. The Developer Survey computes real maturity per dimension, cross-checkable against leadership-declared maturity in the Score Justification PDF.
