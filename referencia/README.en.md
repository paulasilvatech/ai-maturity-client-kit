# `referencia/` — Technical documentation and examples

This folder contains **read-only documentation**: the official algorithm, the description of the 158 questions, the auditable spreadsheet, the interactive calculator and the final output examples.

## Files

### 📊 Scoring algorithm documentation

| File | What it is |
|---|---|
| **[pontuacao-e-calculo.md](pontuacao-e-calculo.md)** | Official scoring reference: all formulas (capability/pillar/overall), threshold (25/40), multi-respondent, gap analysis (P0-P3), label mapping, edge cases, glossary. **Mirrors 1:1** the code in `app/backend/src/scoring.rs` (platform repository, not included in this kit) |
| **[pontuacao-e-calculo.xlsx](pontuacao-e-calculo.xlsx)** | Auditable Excel workbook with 3 tabs: Read-me, Responses (all 158 questions, weights from framework.json) and Calculation (28 capabilities, pillars, overall, threshold). SUMPRODUCT formulas visible cell by cell. Generated by `scripts/generate_scoring_workbook.py` |
| **[calculadora-pontuacao.html](calculadora-pontuacao.html)** | Standalone interactive calculator (Tailwind + JavaScript): 17 real questions (1 capability per pillar), edit levels and weights, watch scores recalculate live. Opens in the browser |

## When to use each file

- **Want to understand how the scores are calculated?** → `pontuacao-e-calculo.md`
- **The client asked "how did this score come out?" and wants to see the formula?** → open `pontuacao-e-calculo.xlsx` in Excel
- **Want to play with different responses and see scores live?** → open `calculadora-pontuacao.html` in the browser
- **About to answer a question and unsure what each level means?** → consult `P1/P2/P3-…md` (more detail than the visual HTMLs in `formularios/`)
- **Want to see "what the final report will look like"?** → open the PDFs in `exemplo-saida/`

## ⚠️ Important

**Do not modify** any file in this folder: they are **read-only** (technical reference that must remain canonical). If you need to customize for your client, copy elsewhere and edit the copy.

The only exception: the `exemplo-saida/` subfolder can be regenerated by running the pipeline with `respostas.json.example`.
